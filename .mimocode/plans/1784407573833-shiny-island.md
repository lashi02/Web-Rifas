# Plan: Frontend Plataforma de Rifas - Web Rifas

## Contexto

Plataforma de rifas/sorteos con reserva de números, aprobación manual de pagos y panel administrativo.
Público: clientes cubanos + familiares en EE.UU. que pagan por Zelle. Uso mayoritario desde celular con datos lentos.
Stack actual: Astro 7 + Tailwind CSS 4 + React 19 + TypeScript + Supabase (pendiente de configurar).

El frontend actual tiene una base genérica (hero simple, grid de rifas, "cómo funciona"). Hay que transformarlo según el informe técnico con estética premium/casino elegante, colores negro-dorado-azul eléctrico, y funcionalidad completa de reserva de números.

## Criterios de Aceptación (MVP)

- [ ] La web abre rápido en celular con datos móviles
- [ ] El contador regresivo funciona y es visible sin scroll
- [ ] La barra de progreso muestra vendidos/total correctamente
- [ ] Los números disponibles, reservados y pagados se distinguen visualmente
- [ ] Un usuario puede reservar números sin crear cuenta larga
- [ ] Los números reservados se bloquean para otros usuarios
- [ ] El flujo genera mensaje/mensaje de WhatsApp correcto
- [ ] La web es 100% responsive en móvil
- [ ] Estética premium oscura con dorados (no parece web de apuestas)
